# Contributing to Chatty

Thanks for your interest in contributing! Here's how to get started.

## How to contribute

1. **Fork** this repository
2. **Clone** your fork locally
3. **Create a branch** for your change (`git checkout -b my-feature`)
4. **Make your changes** and test them locally with `python run.py`
5. **Commit** with a clear message describing what you changed and why
6. **Push** to your fork and open a **Pull Request** against `master`

## Pull request guidelines

- Keep PRs focused — one feature or fix per PR
- Describe what the change does and why in the PR description
- Make sure the app runs locally before submitting
- New integrations should follow the existing pattern in `backend/integrations/`

## Reporting bugs

Open an [issue](https://github.com/WWilson1017/chatty/issues) with:
- What you expected to happen
- What actually happened
- Steps to reproduce
- Your environment (OS, Python version, Node version)

## Feature requests

Open an [issue](https://github.com/WWilson1017/chatty/issues) and describe the use case. Explain the problem you're trying to solve, not just the solution you have in mind.

## Code style

- Python: Follow existing patterns in the codebase
- TypeScript/React: Follow existing patterns in `frontend/src/`
- No need to add comments for self-explanatory code

## Questions?

Open a [discussion](https://github.com/WWilson1017/chatty/discussions) for general questions or ideas.
