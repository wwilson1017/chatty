# BambooHR Setup

Connect your BambooHR account so your Chatty agents can look up employee info, check who's out, and view time-off data.

## Prerequisites

- A BambooHR account with admin access
- Your BambooHR subdomain (the `company` part of `company.bamboohr.com`)

## Step 1: Generate an API Key

1. Log in to BambooHR
2. Click your profile picture in the upper right > **API Keys**
3. Click **Add New Key**
4. Name it (e.g., "Chatty") and click **Generate Key**
5. Copy the API key — you won't be able to see it again

> **Note:** Only BambooHR admins can generate API keys. If you don't see the API Keys option, ask your BambooHR administrator.

## Step 2: Connect in Chatty

### Local

1. Start Chatty with `python run.py`
2. Go to **Settings** > **Integrations**
3. Find **BambooHR** and click **Setup**
4. Fill in the connection form:
   - **Subdomain** — your BambooHR subdomain (e.g., if your URL is `acme.bamboohr.com`, enter `acme`)
   - **API Key** — the key you generated in Step 1
5. Click **Connect** — Chatty tests the connection by pulling your employee directory

### Railway

1. Open your Chatty instance on Railway
2. Go to **Settings** > **Integrations**
3. Find **BambooHR** and click **Setup**
4. Fill in the same connection form (Subdomain, API Key)
5. Click **Connect** — Chatty tests the connection by pulling your employee directory

BambooHR credentials are entered in the Chatty UI, not as environment variables — so the setup is the same on both local and Railway.

## What Your Agents Can Do

Once connected, your agents can:

- **Look up employees** — search by name, view profiles with title, department, email, phone
- **Browse the directory** — list employees, filter by department
- **Check who's out** — see who's on PTO today or any date range
- **View time-off requests** — list pending, approved, or denied requests
- **Check PTO balances** — see remaining time-off for any employee

Example questions you can ask:

- "Who's out of the office today?"
- "Look up Jane Smith's contact info"
- "How many PTO days does John have left?"
- "Show me everyone in the engineering department"
- "Are there any pending time-off requests this month?"

## Notes

- BambooHR access is **read-only** by default — your agent can look up data but won't modify employee records through the standard tools
- Employee name lookups use fuzzy matching, so you don't need to type exact names
- API keys are generated by admins only — regular employees can't create them
- Credentials are encrypted at rest in your Chatty instance
